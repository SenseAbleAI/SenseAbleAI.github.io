# SenseAble

SenseAble is a research-driven system for **personalized, culturally aware, and multisensory text accessibility**. Many everyday texts rely on sensory descriptions such as color, sound, smell, taste, touch, or metaphor, which can unintentionally exclude readers with sensory impairments, neurodivergent processing styles, or unfamiliar cultural backgrounds. SenseAble detects these sensory-dependent phrases, explains them, and rewrites them when needed in a **meaning-preserving and personalized** way to improve clarity and inclusion. 

🌐 **Check out the demo:** https://senseableai.github.io/login

## How SenseAble Works

1. A user pastes text into the editor  
2. SenseAble detects sensory-rich or culturally grounded phrases  
3. Detected phrases are tagged and explained  
4. Users review or modify tags to personalize the system  
5. SenseAble generates accessible rewrite options based on confirmed preferences

## Key Features

- 🎯 **Personalization-first design**  
  Adapts text based on accessibility needs, cultural context, and evolving user preferences.

- 🏷️ **Sensory-aware tagging**  
  Automatically detects vision-, sound-, smell-, taste-, touch-, and cross-sensory phrases with short, clear explanations.

- 🔁 **Meaning-preserving rewrites**  
  Generates gentle to fully literal rewrites without removing intent or nuance.

- ✏️ **Interactive refinement**  
  Users can modify detected tags to fine-tune how the system understands and rewrites text.


## Accessibility-Focused Interface

- Accessible tag colors using the Okabe–Ito color palette for color-blind users  
- Dyslexia-friendly tag styling and formatting  
- One-page UI to reduce cognitive load  
- Editable user profile for updating preferences anytime
